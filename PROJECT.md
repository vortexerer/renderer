# 🛠️ 프로젝트 상황판: C++20/DirectX 12/OpenXR 기반 VR 게임 엔진 자체 개발

본 프로젝트는 외부 상용 엔진(Unity, Unreal)에 의존하지 않고, **C++20**, **DirectX 12 (Direct3D 12)**, **OpenXR** 표준 하드웨어 API를 사용하여 실시간 3D 가상현실 공간을 제어하는 **VR 게임 개발 엔진(VR Game Engine)**의 핵심 코어를 바닥부터 직접 설계 및 제작하는 것을 목표로 합니다.

개발된 모든 소스 코드는 엔진의 핵심 요소들(기하 변환 수학, 렌더링 파이프라인, Cartesian 물리 연산, WASAPI 입체 오디오, glTF 바이너리 파서, 3D 공간 UI)이 설계 스펙대로 오차 없이 물리적/수학적 정합성을 유지하며 완벽하게 기능하는지 검증하기 위한 **E2E 자동 테스트 프레임워크**를 탑재하고 있습니다.

## 🏗️ 엔진 서브시스템 아키텍처 (Engine Architecture)
엔진 코어는 모듈 간의 결합도를 차단하고 독립적인 재사용성을 확보하기 위해 다음의 **8대 독자 서브시스템**으로 구조화되었습니다.

1. **Core & Platform (OS 및 플랫폼 추상화)**
   - Win32 API 기반 창 제어 및 윈도우 루프 관리.
   - `QueryPerformanceCounter` 기반 마이크로초 단위 타이머 구현.
   - 멀티스레드 환경에서 동작하는 비동기 링 버퍼 파일 로깅 시스템.
2. **Math Library (기하 수학 모듈 - 교과 기하 수학 연계)**
   - Cartesian 좌표 기반 3차원 벡터(`Vector3`) 연산자 오버로딩.
   - 4x4 아핀 변환 행렬 및 비균등 스케일링 보정용 법선 역전치 행렬(`Matrix4x4`).
   - Hamilton 사원수(`Quaternion`) 및 SLERP(구면 선형 보간) 연산 모듈.
3. **Renderer (Direct3D 12 그래픽스 엔진)**
   - DX12 Pipeline State Object(PSO) 및 Root Signature 직접 캡슐화.
   - GPU 멀티패스 비디오 메모리 리소스 배리어 최적화 관리.
   - Instanced Stereo Rendering(양안 동시 투사) 기법 구현.
4. **OpenXR Manager (VR 런타임 제어)**
   - OpenXR API 바인딩을 통한 헤드셋(HMD) 및 좌우 컨트롤러 포즈(Pose) 수동 트래킹.
   - 입체 시야 확보를 위한 스테레오 프로젝션 행렬 수집.
5. **Physics Engine (Cartesian 물리 엔진 - 고급물리 연계)**
   - Cartesian Rigid Body 역학 상태(변위 $\vec{r}$, 속도 $\vec{v}$, 가속도 $\vec{a}$) 제어.
   - 프레임률 편차 보정 및 시뮬레이션 안정성을 위한 **Fixed Timestep Accumulator** 동기화.
   - 에너지 보존형 **심플렉틱 반암시적 오일러(Semi-implicit Euler)** 이산 적분기.
   - Minkowski Difference 기반 GJK(Simplex) 및 EPA(Polytope) 3D 정밀 충돌/관통 해결 및 Impulse 물리 반응 연산.
6. **Audio Engine (WASAPI 공간 음향 엔진 - 파동물리 연계)**
   - WASAPI Exclusive Mode(배타적 모드) 직접 제어를 통한 지연 시간(Latency) 제로 구현.
   - 소리와 청취자 공간 벡터 연계 시간차(ITD) 및 음량 감쇠(ILD) 머리전달함수(HRTF) 공간 합성.
   - 도플러 효과(Doppler Effect) 주파수 이동 물리 연산 및 피치 변조.
7. **Asset Loader (에셋 파이프라인 수동 파서)**
   - 외부 라이브러리(Assimp, TinyglTF 등) 없이 glTF 2.0 JSON 파서 및 바이너리 `.bin` 버퍼 디코더 자체 구현.
   - BMP 이미지 포맷의 바이너리 파일 헤더 수동 언패커.
8. **UI System (3D 월드 공간 Immediate UI)**
   - 오프스크린 렌더 타깃 UI 렌더러 구현.
   - 3D 공간 상의 Quad 텍스처 투사 및 레이캐스트(Raycast) 물리 교차점 UI 상호작용.

## 🎓 생활기록부(세특) 기재용 최종 정리 템플릿 (School Record Templates)
고등학교 생활기록부(진로활동 및 과목별 세특)에 기재 시 그대로 복사-붙여넣기 하거나 참고할 수 있도록 구성된 완성형 학술 요약 문구입니다. (글자 수 및 학술 용어 최적화 완료)

### 📌 1. 진로활동 (컴퓨터공학, 시스템 소프트웨어)
> **[주제: 상용 엔진 구조 분석을 통한 자체 게임 엔진 아키텍처 설계 및 저수준 하드웨어 제어]**
> 상용 게임 엔진의 높은 추상화 수준을 넘어, 시스템 설계자(System Architect) 관점에서 저수준 컴퓨터 아키텍처와 운영체제(OS)와의 상호작용 방식을 깊이 탐구함. C++20 멀티스레드 기반 비동기 로깅 시스템을 직접 설계하고, GPU 하드웨어 제어(DirectX 12 API), VR 디바이스 인터페이스(OpenXR API), Windows 오디오 버퍼 직접 제어(WASAPI Exclusive Mode) 등 저수준 시스템 프로그래밍 영역을 직접 코딩하여 모듈 간 결합도가 낮은 독자적인 8대 엔진 서브시스템 아키텍처를 성공적으로 구현함. 또한 Mock 시뮬레이션 환경을 구축하여 17개의 시나리오를 자동 검증하는 E2E 테스트 인프라를 실증하는 등 우수한 소프트웨어 엔지니어링 설계 역량을 증명함.

### 📌 2. 고급수학Ⅰ (기하 변환, 사원수)
> **[주제: 3D 오일러 회전 변환 행렬의 특이점(Gimbal Lock) 수학적 논증 및 Hamilton 사원수 구면 선형 보간(SLERP) 구현]**
> 3차원 공간 내 회전 변환에서 아핀 행렬 곱셈의 비가환성(식 4.17)을 학습하고, 피치(Pitch)각이 $90^\circ$에 도달하는 특이 한계에서 코사인 성분 소실로 인해 헤드(Yaw)와 롤(Roll)이 단일 합산각으로 묶이는 짐벌락(Gimbal Lock) 현상을 수학적으로 엄밀히 유도함(식 4.21 연계). 이 회전 차원 실종 한계를 해결하기 위해 실수부와 허수부로 구성된 4차원 Hamilton 사원수 곱(식 4.32)과 두 회전 사원수 사이를 등속 최단 대원으로 이동시키는 구면 선형 보간(SLERP, 식 4.53) 연산 공식을 C++ 클래스로 구현함. 또한, 비균등 스케일 변환 시 표면 법선 벡터의 수직 기하 관계를 보존하기 위해 역의 전치 행렬(Transpose of Inverse) $A = (M^{-1})^T$ 연산을 행렬 대수학적으로 도출하여 물리적 모순을 방지하는 등 수준 높은 기하 대수 해석 역량을 입증함.

### 📌 3. 고급물리학 (컴퓨터 물리학, 고전역학 이산화)
> **[주제: Cartesian 강체 역학의 이산화 수치 적분 안정성 확보를 위한 반암시적 오일러 적분 및 공간 음향 파동학 융합]**
> Newton의 제2법칙 미분 방정식을 Cartesian 3차원 좌표계 상의 변위($\vec{r}$), 속도($\vec{v}$), 가속도($\vec{a}$) 벡터 데이터로 모형화함. 컴퓨터 성능 차이로 인한 가변 델타 타임이 물리 시뮬레이션의 터널링과 무한 에너지 발산 오류를 야기함을 규명하고, 물리 연산 루프를 100Hz로 격리하는 고정 시간 누적기(Fixed Timestep Accumulator) 시스템을 설계함. 일반 오일러 적분 대비 위상 공간 내 에너지가 보존되는 심플렉틱 적분식인 반암시적 오일러(Semi-implicit Euler) 공식을 유도 및 코딩하여 물리적 안정성을 입증하였으며, GJK/EPA 3D 충돌 알고리즘을 C++로 구축함. 또한 파동물리학의 소리 전달 시간차를 컴퓨터 상에 구현하기 위해 양이 시간차(ITD)와 Low-Pass 필터를 통한 음량차(ILD) HRTF 공간 음향 공식과 도플러 효과 피치 변조 공식을 자체 사운드 믹서 스레드에 성공적으로 이식하여 융합 학문적 역량을 과시함.
